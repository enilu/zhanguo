# Claude 工具入口文件

## 执行顺序
1. 先读 `README.md` 关键章节
2. 扫描并命中 `doc/ai-skills/` 的 `SKILL.md`
3. 最后读本文件差异项

## 与 README 冲突时的处理原则
与 `README.md` 冲突时，以 `README.md` 为准。

## 专属差异项

### 工具特性
- 支持 Plan mode，规划阶段只输出方案，不直接写正文
- 具备较强的文档理解和内容生成能力
- 对长文本处理能力优秀

### 协作与交互约束
- 文档内容与注释默认使用中文
- 在编写任何正文内容前，先描述写作思路并等待用户批准
- 用户需求存在歧义时，先提出澄清问题
- 预估需要修改超过 3 个文件时，先暂停并拆分成更小子任务
- 每轮交互结束后，将本轮内容追加写入 `session_yyyyMMdd.md`
- 最终回复前，必须执行 `tail -n 20 session_yyyyMMdd.md` 做可见校验
- 最终输出必须包含：`会话落盘：已完成/失败 + 文件路径 + 时间戳`

### 通用实施与输出规范
- 搜索与定位优先使用 `rg` / `rg --files`
- 改动保持最小化，优先复用现有模式
- 较大改动先给简短计划，再分步落地
- 验证至少覆盖与改动直接相关的核心内容，无法验证时明确说明原因
- 涉及设定、情节、人物、结构或前后文联动时，必须显式说明影响范围
