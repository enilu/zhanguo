# 战国小说项目

这是一个以中国战国时期为背景的长篇历史演义小说项目。
探索下使用ai独立写长篇小说的可能性（结论：不太行┭┮﹏┭┮）
项目当前目标是：

- 以 `秦国东出，终成一统` 为主线
- 采用 `天下共视角`
- 起笔于 `苏秦张仪时代`
- 风格定位为 `史实主干 + 演义化适中`
- 强化人物能力辨识度与讨论度，使读者在读完后愿意讨论：
  - 名将排名
  - 统帅排名
  - 纵横家排名
  - 悲剧人物排名

## 写作原则

- 不把秦写成天然正义的一方
- 不把六国写成整齐划一的昏庸群像
- 不把战国写成三国翻版
- 不靠夸张单挑支撑武将魅力
- 重点写出：
  - 纵横
  - 统兵
  - 国策
  - 国运
  - 悲剧英雄

## AI Agent Guide（统一入口）

### 执行顺序
1. 先读 README 关键章节
2. 扫描并命中 `doc/ai-skills/` 的 `SKILL.md`
3. 最后读工具入口差异项

### 协作与交互约束（高优先级）
- 文档内容与注释默认使用中文
- 编写任何正文内容前，先描述写作思路并等待用户批准
- 需求有歧义时，先澄清再实施
- 预估改动超过 3 个文件时，先拆分子任务再继续
- 若支持 Plan mode，则规划阶段只输出方案
- 每轮交互结束后，追加写入 `session_yyyyMMdd.md`
- 最终回复前，必须先完成会话落盘，并执行 `tail -n 20 session_yyyyMMdd.md` 校验
- 最终输出必须包含：`会话落盘：已完成/失败 + 文件路径 + 时间戳`

### 通用实施与输出规范
- 搜索与定位优先使用 `rg` / `rg --files`
- 改动保持最小化，优先复用现有模式
- 较大改动先给简短计划，再分步落地
- 验证至少覆盖最小相关内容，无法验证时明确说明原因
- 最终输出至少包含关键改动文件、改动目的、验证结果、未覆盖风险/待确认项、影响范围
