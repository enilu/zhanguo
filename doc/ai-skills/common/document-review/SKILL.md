# 文档审读 Skill

## 何时使用
- 对章节草稿进行审读时
- 检查设定一致性时
- 评估大纲质量时
- 任何需要对文档内容进行审查的任务

## 输入前置
- 已读 `STORY_BIBLE.md`
- 已读相关的 `docs/` 文件

## 硬规则

### 审读原则
- 必须遵循项目核心约束
- 重点检查内容与设定的一致性
- 评估内容是否符合项目风格要求
- 识别可能影响项目质量的问题

### 输出要求
- 审读报告必须带时间戳
- 报告应包含具体问题和改进建议
- 报告必须放入 `reviews/records/` 目录

## 执行步骤

先阅读 `references/review-workflow.md` 查看详细流程。
遇到高频坑时查看 `references/gotchas.md`。
若存在 `scripts/`，优先执行脚本化检查。
