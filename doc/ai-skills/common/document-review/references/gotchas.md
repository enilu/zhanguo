# 文档审读常见坑点

## 高频错误场景

### 1. 设定一致性问题
- **错误表现**：人物能力不符合设定、事件时间线混乱
- **后果**：影响作品的可信度和连贯性
- **正确做法**：严格对照项目设定文档进行检查

### 2. 内容质量问题
- **错误表现**：文字表达不清晰、情节设计不合理、人物塑造模糊
- **后果**：降低作品的可读性和吸引力
- **正确做法**：注重文字表达、情节设计和人物塑造的质量

### 3. 结构完整性问题
- **错误表现**：目录不完整、章节结构混乱、段落划分不合理
- **后果**：影响作品的可读性和系统性
- **正确做法**：注重文档结构的完整性和规范性

## 验证点

### 内容验证
1. 检查是否符合项目设定
2. 检查内容质量是否符合要求
3. 检查结构是否完整

### 格式验证
1. 文件名是否符合现有模式
2. 文档结构是否清晰
3. 语言是否符合项目要求

## 处理方法

### 发现问题时的处理流程
1. 确认问题是否真实存在
2. 定位问题所在文件和位置
3. 分析问题对项目的影响程度
4. 提出修改方案
5. 等待用户批准后实施
6. 验证修改是否符合要求

### 不确定时的处理方法
1. 先保留原始内容
2. 提出澄清问题
3. 等待用户确认后再进行修改

## 工具使用注意事项

### 自动审读脚本
- 使用 `bash scripts/auto-review.sh` 进行自动审读
- 检查脚本语法是否正确：`bash -n scripts/auto-review.sh`
- 确保脚本的输入和输出格式符合要求

### 手动审读
- 阅读文档时要逐字逐句检查
- 对照项目设定文档进行验证
- 记录发现的问题和改进建议
