# 文档审读流程

## 审读类型

### 1. 设定一致性审读
- **目标**：检查内容与项目设定是否一致
- **范围**：人物设定、事件时间线、地理信息等
- **依据**：`STORY_BIBLE.md`、`docs/characters/`、`docs/outlines/` 等

### 2. 内容质量审读
- **目标**：评估内容的质量和可读性
- **范围**：文字表达、情节设计、人物塑造等
- **依据**：项目写作原则

### 3. 结构完整性审读
- **目标**：检查文档结构是否完整
- **范围**：目录、章节、段落结构等
- **依据**：项目文档规范

## 审读流程

### 1. 准备阶段
1. 确定审读类型和范围
2. 收集相关参考文档
3. 准备审读工具和模板

### 2. 执行阶段
1. 阅读待审读文档
2. 逐项检查内容
3. 记录发现的问题
4. 评估问题的严重程度
5. 提出改进建议

### 3. 报告阶段
1. 整理审读结果
2. 生成审读报告
3. 保存到 `reviews/records/` 目录
4. 通知相关人员

## 审读内容

### 设定一致性检查
- 人物能力是否符合设定
- 事件时间线是否准确
- 地理信息是否正确
- 社会制度是否符合历史背景
- 文化习惯是否合理

### 内容质量检查
- 文字表达是否清晰准确
- 情节设计是否合理
- 人物塑造是否鲜明
- 主题表达是否明确
- 阅读体验是否良好

### 结构完整性检查
- 目录是否完整
- 章节结构是否清晰
- 段落划分是否合理
- 标题层级是否规范
- 引用和参考是否正确

## 问题分类

### 严重问题
- 违反项目核心约束
- 与历史事实严重不符
- 内容质量极差
- 结构严重混乱

### 一般问题
- 与设定有细微不符
- 文字表达不够流畅
- 情节设计有小缺陷
- 结构有小问题

### 建议性问题
- 内容可以进一步优化
- 文字可以更生动
- 情节可以更紧凑
- 结构可以更清晰

## 处理方法

### 严重问题
1. 立即停止审读
2. 通知项目负责人
3. 待问题解决后重新审读

### 一般问题
1. 记录问题和建议
2. 提交给文档作者
3. 待修改后重新检查

### 建议性问题
1. 记录建议
2. 与作者沟通
3. 确定是否需要修改
