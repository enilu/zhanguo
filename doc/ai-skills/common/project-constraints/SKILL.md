# 项目约束检查 Skill

## 何时使用
- 进行正文写作时
- 修改人物设定时
- 修改大纲时
- 进行审读时
- 任何可能影响项目核心设定的任务

## 输入前置
- 已读 `README.md` 关键章节
- 已读 `STORY_BIBLE.md`

## 硬规则

### 项目核心不变量
- 主线：`秦国东出，终成一统`
- 立场：`天下共视角`
- 起点：`苏秦张仪时代`
- 篇幅：`90 回左右`
- 结构：`纵横家开局，名将收束，帝王定局`
- 语言：`现代白话为主，关键处带古意`
- 风格：`史实主干 + 演义化适中`

### 写作原则
- 不把秦写成天然正义的一方
- 不把六国写成整齐划一的昏庸群像
- 不把战国写成三国翻版
- 不靠夸张单挑支撑武将魅力
- 重点写出：纵横、统兵、国策、国运、悲剧英雄

## 执行步骤

先阅读 `references/project-constraints.md` 查看详细约束。
遇到高频坑时查看 `references/gotchas.md`。
若存在 `scripts/`，优先执行脚本化检查。
