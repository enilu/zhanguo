# 项目约束检查常见坑点

## 高频错误场景

### 1. 秦国塑造过度
- **错误表现**：把秦国写成天然正义、道德高尚的一方
- **后果**：削弱了统一过程的复杂性和历史深度
- **正确做法**：突出秦国的制度优势、战略规划和执行力，但也要展示其残酷性和局限性

### 2. 六国形象单一
- **错误表现**：把六国全部写成昏庸无能、腐败不堪的群像
- **后果**：使统一过程显得过于容易，缺乏戏剧性
- **正确做法**：每个国家要有不同的失败原因和可敬人物

### 3. 武将形象三国化
- **错误表现**：过度依赖单挑、夸大个人武力
- **后果**：偏离了战国时代的战争逻辑
- **正确做法**：突出统帅的战略眼光、治军能力和战场指挥艺术

### 4. 历史事件时间线混乱
- **错误表现**：事件发生顺序不符合历史记载
- **后果**：影响作品的可信度
- **正确做法**：严格遵循历史时间线，仅在必要时进行合理虚构

### 5. 人物能力标签模糊
- **错误表现**：人物能力线不清晰，缺乏独特标签
- **后果**：人物形象不够鲜明，读者难以记住和讨论
- **正确做法**：每个核心人物要有明确的能力标签和代表场景

## 验证点

### 内容验证
1. 检查是否符合项目核心不变量
2. 检查人物塑造是否符合能力体系
3. 检查事件时间线是否准确
4. 检查是否避免了必须避免的写法

### 格式验证
1. 文件名是否符合现有模式
2. 文档结构是否清晰
3. 语言是否符合项目要求

## 处理方法

### 发现问题时的处理流程
1. 确认问题是否真实存在
2. 定位问题所在文件和位置
3. 分析问题对项目的影响程度
4. 提出修改方案
5. 等待用户批准后实施
6. 验证修改是否符合要求

### 不确定时的处理方法
1. 先保留原始内容
2. 提出澄清问题
3. 等待用户确认后再进行修改
