# 章节正文写作约束

这份文档用于约束 `docs/chapter-XX-draft.md` 的正文写法，重点解决以下常见问题：

- 正文混入审核/备注/承接信息
- 结构判断太多，场面太少
- 叙述者替读者过度解题
- 多个核心人物声口趋同
- 章末像总结，不像小说收束

## 一、正文边界

- 章节正文只保留小说内容。
- 不要在正文中加入 `本回小结`、审核说明、写作备注、承接提示、下一回提示。
- 审读内容只放进 `reviews/records/`。
- 如确需工作备注，单独建正文之外的文件存放。

## 二、场面优先

- 一回里若既有“事件”又有“判断”，应让事件承担主要叙事重量。
- 能用动作、停顿、视线、军令、器物、空间调度表现的，不要先用解释句说破。
- 少写“这句话为什么毒”“这一层真正厉害在哪”，先让人物和局面自己显出来。

## 三、避免评论稿化

- 不要把章节写成战略分析稿、结构总结稿、卷末评论稿。
- 少用连续抽象判断句堆出“中心思想”。
- 同一判断不要在一回内反复换说法重复三次以上。

## 四、句式重复控制

以下句式可以用，但不能高频霸屏：

- `不是……而是……`
- `到这一步`
- `真正……`
- `他知道……`
- `这一句/这一层/这一步`

如果连续几段都在“解释其意义”，就应回到场面本身重写。

## 五、人物声口区分

- 张仪：短，稳，冷，刀口藏在平话里。
- 苏秦：宽，沉，有悲意，但要系在具体物和动作上。
- 屈原：直，硬，重是非，不替人找好听说法。
- 昭阳：重军政执行，重后果，少空谈。
- 陈轸：像诊病开方，专拆病根。
- 楚怀王：先护脸面，再算利害，摇摆时要有人的急气。

## 六、章末收束

章末优先落在具体动作或未完之势上，例如：

- 命令刚下
- 信已写成但未发
- 人心已偏但未认
- 器物被拿起、放下、收起
- 一场军政变化刚露出后果

不要轻易用“这一段说明了……”式结尾直接盖棺。
